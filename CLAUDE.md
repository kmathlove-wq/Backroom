# CLAUDE.md

## Project Context

- Static Backrooms web experience.
- Main files:
  - `index.html`: page shell, favicon, Three.js import map, script loading.
  - `sytle.css`: full-screen canvas, HUD, crosshair, vignette/noise overlay.
  - `main.js`: Three.js scene, generated textures, tile generation, movement, collision, lights, fake shadows.
  - `backroom-favicon-2026.svg`: unique favicon path to avoid cache/name collisions.
- No package manager setup is required for the app itself.

## Known Rendering Constraints

- Real WebGL shadows and `MeshStandardMaterial` previously made the scene render incorrectly in this environment.
- Keep the reliable approach: visible `MeshBasicMaterial`, canvas-generated textures, and soft transparent shadow textures.
- The user prefers realistic Backrooms mood, but visibility is more important than physically correct lighting.
- Fake wall shadows are generated by `makeSoftWallShadowTexture()`.
- Fake column shadows are generated by `makeSoftColumnShadowTexture()`.

## Validation

- Run `node --check main.js` after JavaScript edits.
- If a local server is needed, use `python3 -m http.server 8000`.
- Check static assets with `curl -I http://127.0.0.1:8000/<file>` when useful.
